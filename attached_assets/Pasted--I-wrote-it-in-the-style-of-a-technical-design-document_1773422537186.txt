Online Cremation System
Automated End-to-End Cremation Workflow
Version: 1.0
Purpose: System Architecture + Operational Design
Scope: Online cremation order intake, authorization, payment, logistics handoff, and automated compliance packet generation.

1. System Goal
The goal of this system is to allow a family to complete an online cremation arrangement from anywhere while ensuring that the funeral home maintains a complete legal and compliance record with minimal staff intervention.
The system must accomplish three major objectives:
Provide a simple and respectful online experience for families.


Minimize staff workload and administrative overhead.


Automatically generate a complete compliance packet for regulatory and legal protection.


The final result should be a system where staff only perform essential operational tasks while the platform automatically handles documentation, record keeping, and audit preparation.

2. Core Design Principles
Automation First
All documentation and compliance artifacts should be generated automatically as the case progresses.
Staff should never have to manually assemble compliance documentation.

Minimal Sensitive Data Storage
The system should avoid storing regulated or sensitive data whenever possible.
Key principle:
• Google Drive becomes the document system of record
• The application stores references and metadata only
Sensitive documents should exist in external storage rather than inside the application database.

Event-Driven Workflow
The system should operate using domain events instead of manual checklists.
Examples of events:
ORDER_CREATED
FORM_SUBMITTED
PAYMENT_CONFIRMED
REMAINS_RECEIVED
CREMATION_COMPLETED
SHIPMENT_RECORDED
CASE_COMPLETED
Each event automatically triggers downstream actions like document generation and compliance packet updates.

Compliance by Design
Every case must produce a complete legal record that proves:
• cremation was authorized
• identity and authority were verified
• remains were tracked properly
• documents were preserved
• payment and release were recorded
This protects the funeral home in the event of audits, disputes, or regulatory inspections.

3. System Architecture
The platform is composed of several integrated systems.
Public Website (Replit App)
        |
Order API
        |
Application Database (metadata only)
        |
Jotform (forms + signatures)
        |
Google Drive (document storage)
        |
Payment Provider (Stripe / Square)
        |
Logistics Partner
Each component serves a specific role.

Replit Application
Responsible for:
• public online cremation intake
• staff dashboard
• workflow orchestration
• event logging
• compliance packet automation
The application stores only safe metadata and references.

Jotform
Responsible for:
• collecting family information
• capturing legal authorization signatures
• gathering vital statistics
• collecting SSN only when required
Jotform exports PDFs directly to Google Drive.

Google Drive
Acts as the official document repository.
All documents generated by the system are stored in Drive using a standardized folder structure.

Payment Provider
Handles hosted payment processing.
The application stores only:
• payment status
• transaction reference
• payment timestamp
The application never stores card data.

4. Workflow Phases
The system is organized into sequential phases.
Each phase corresponds to a major milestone in the cremation process.

Phase 1 — Public Online Cremation Start
Goal
Allow families to begin the cremation process online using a simple form.
This step should collect only minimal information.
Required fields:
contact_name
contact_email
contact_phone
decedent_name
state_of_death
Once submitted, the system generates:
• an Order record
• a random order_token
• a case folder structure in Google Drive

Why This Phase Matters
Collecting minimal information reduces friction for families and avoids collecting sensitive information too early in the process.
The order token becomes the secure reference used throughout the workflow.

Phase 2 — Forms and Legal Authorization
Goal
Collect all required legal documentation.
Forms are managed through Jotform.
Required forms include:
Online Cremation Intake
Cremation Authorization
Vital Statistics
Optional SSN Collection
Shipping / Urn Personalization
Each form submission automatically exports a PDF to Google Drive.

Security Requirement
Sensitive data must never appear in URLs.
Prefill links should contain only:
order_token
serviceType
safe contextual fields

Why This Phase Matters
Cremation authorization is the most important legal requirement.
Proper documentation ensures that the funeral home can demonstrate that the cremation was legally authorized.

Phase 3 — Payment
Goal
Collect payment using a hosted payment page.
Staff send a payment link after required forms are completed.
Payment providers may include:
• Stripe Checkout
• Square Checkout
• Hosted invoice link

Why This Phase Matters
Using hosted payment keeps the platform outside of full PCI compliance scope and reduces cybersecurity risk.
The system stores only the transaction reference.

Phase 4 — Operational Fulfillment
Goal
Track the operational stages of the cremation process.
Key events:
REMAINS_RECEIVED
CREMATION_SCHEDULED
CREMATION_COMPLETED
RELEASE_RECORDED
SHIPMENT_RECORDED
Each event is automatically logged in the system.

Why This Phase Matters
These events create a chain-of-custody log that proves the remains were handled properly.
This log becomes a critical part of the compliance packet.

Phase 5 — Compliance Packet Automation
Goal
Automatically assemble a complete legal record for the case.
The compliance packet includes:
Case Summary
Cremation Authorization
Vital Statistics
Chain of Custody Log
Payment Record
Release / Shipping Record
SSN documents are stored separately in a restricted folder.

Why This Phase Matters
Regulators and funeral boards often require these records.
Automatic packet generation eliminates the need for staff to manually compile documents.

5. Google Drive Folder Structure
Each case uses a standardized folder structure.
Orders/
  {OrderToken}/

    01 Intake
    02 Authorization
    03 Vital Stats
    04 SSN Restricted
    05 Shipping & Urn
    06 Compliance Packet

6. Compliance Packet Contents
The final packet should include the following documents.
Case_Summary.pdf
Cremation_Authorization.pdf
Vital_Statistics.pdf
Chain_of_Custody_Log.pdf
Payment_Record.pdf
Release_or_Shipping_Record.pdf
Final_Compliance_Packet.pdf
The final compliance packet merges all non-restricted documents into a single file.

7. Audit Trail
Every action within the system should generate an audit event.
Example events:
Form link sent
Form submitted
Authorization signed
Payment confirmed
Remains received
Cremation completed
Shipment recorded
Case closed
Audit logs protect the funeral home if a case is questioned later.

8. Compliance Lock
Once a case reaches completion and the final compliance packet is generated:
packet_locked = true
After locking:
• documents cannot be modified
• new events create addendum entries
• the original packet remains unchanged
This protects the legal record from accidental edits.

9. Staff Interaction Model
Staff should only perform the following actions:
send forms
confirm remains received
mark cremation completed
record shipment or release
Everything else should occur automatically.
This dramatically reduces administrative workload.

10. Benefits of the System
Implementing this system provides several advantages.
Operational Efficiency
Staff spend less time managing paperwork.

Legal Protection
The funeral home always has a complete compliance record.

Family Experience
Families can complete arrangements online without unnecessary delays.

Audit Readiness
Compliance packets can be produced instantly during inspections.

11. Future Expansion
The architecture is designed to support future improvements.
Potential extensions include:
• logistics partner API integration
• SMS notifications
• automated coroner release workflows
• state-specific compliance automation
• additional disposition services beyond cremation

12. Summary
The Online Cremation System is designed to combine automation, compliance, and operational efficiency.
By structuring the workflow around events and automatically generating documentation, the platform ensures that every cremation case produces a complete legal record with minimal staff involvement.
This approach allows funeral homes to provide modern online services while maintaining the high level of regulatory compliance required by the industry.
